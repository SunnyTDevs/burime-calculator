# Инструкция по запуску модульных тестов в IntelliJ IDEA

## Проблема с Gradle и SSL

При запуске тестов через командную строку `./gradlew test` может возникнуть ошибка, связанная с SSL сертификатами:
```
Got SSL handshake exception during request
```

Это связано с корпоративными сертификатами или настройками сети.

## ✅ РЕШЕНИЕ: Запуск через IntelliJ IDEA

IntelliJ IDEA имеет встроенный механизм работы с сертификатами и сможет автоматически загрузить зависимости.

### Шаг 1: Синхронизация Gradle проекта

1. Откройте проект в IntelliJ IDEA
2. В правом верхнем углу найдите значок Gradle (слоновик)
3. Нажмите кнопку "Reload All Gradle Projects" (круговая стрелка)

   **ИЛИ**

   Откройте файл `build.gradle` → появится всплывающее окно → нажмите "Load Gradle Changes"

4. Дождитесь завершения загрузки зависимостей (в нижней части IDE будет индикатор прогресса)

### Шаг 2: Запуск всех тестов

**Вариант А: Через файл теста**
1. Откройте файл `src/test/java/com/burime/calculator/BubbleSortTest.java`
2. Нажмите на зеленую кнопку ▶ рядом с именем класса `BubbleSortTest`
3. Выберите "Run 'BubbleSortTest'"

**Вариант Б: Через контекстное меню**
1. В дереве проекта (слева) найдите папку `src/test/java/com/burime/calculator`
2. Щелкните правой кнопкой мыши на `BubbleSortTest.java`
3. Выберите "Run 'BubbleSortTest'"

**Вариант В: Горячие клавиши**
1. Откройте файл `BubbleSortTest.java`
2. Нажмите:
   - **Windows/Linux**: `Ctrl + Shift + F10`
   - **macOS**: `⌘ + Shift + R`

### Шаг 3: Запуск отдельного теста

1. Откройте файл `BubbleSortTest.java`
2. Найдите нужный тест (например, `sortTestNormalArrayWithUniqueElements`)
3. Нажмите на зеленую кнопку ▶ рядом с методом теста
4. Выберите "Run 'sortTestNormal...'"

### Шаг 4: Просмотр результатов

После запуска тестов внизу откроется панель с результатами:

- ✅ **Зеленая галочка** - тест пройден
- ❌ **Красный крестик** - тест провален
- ⚠️ **Желтый треугольник** - тест пропущен

**Статистика:**
- Показывается количество пройденных/проваленных тестов
- Время выполнения каждого теста
- Общее время выполнения всех тестов

## Просмотр покрытия кода тестами

1. Щелкните правой кнопкой мыши на `BubbleSortTest.java`
2. Выберите "More Run/Debug" → "Run 'BubbleSortTest' with Coverage"
3. После выполнения откроется отчет о покрытии кода

## Отладка тестов

Если тест провалился и нужно разобраться почему:

1. Установите точку останова (breakpoint) в нужном месте теста или в тестируемом коде
   - Кликните слева от номера строки
2. Щелкните правой кнопкой на тесте
3. Выберите "Debug 'название_теста'"
4. Используйте панель отладки для пошагового выполнения:
   - **F8** - Step Over (следующая строка)
   - **F7** - Step Into (войти в метод)
   - **Shift + F8** - Step Out (выйти из метода)

## Дополнительные возможности

### Повторный запуск упавших тестов
1. В панели результатов нажмите на иконку "Rerun Failed Tests" (две стрелки)

### Экспорт результатов
1. В панели результатов нажмите на иконку "Export Test Results"
2. Выберите формат (HTML, XML, JSON)
3. Сохраните отчет

### Сравнение ожидаемого и фактического результата
Если тест провалился:
1. В панели результатов выберите проваленный тест
2. Кликните на ссылку "Click to see difference"
3. Откроется визуальное сравнение

## Альтернативный способ: Gradle через IDEA Terminal

Если вы хотите использовать Gradle, но через терминал IntelliJ IDEA:

1. Откройте встроенный терминал: `View → Tool Windows → Terminal`
2. Выполните команду:
   ```bash
   ./gradlew test
   ```

IntelliJ IDEA автоматически использует свои настройки сертификатов, поэтому ошибка SSL не возникнет.

## Решение проблем

### Если зависимости не загружаются

1. **File → Settings** (Windows/Linux) или **IntelliJ IDEA → Preferences** (macOS)
2. **Build, Execution, Deployment → Build Tools → Gradle**
3. Убедитесь, что выбрано:
   - **Build and run using**: Gradle
   - **Run tests using**: Gradle
   - **Gradle JVM**: Project SDK (Java 17)

### Если тесты не запускаются

1. Проверьте, что проект использует Java 17:
   - **File → Project Structure → Project → SDK**: 17
2. Пересоберите проект:
   - **Build → Rebuild Project**

### Если показывает "No tests found"

1. Удалите папку `.gradle` в корне проекта
2. В IDEA: **File → Invalidate Caches → Invalidate and Restart**
3. После перезапуска синхронизируйте Gradle проект
