# Лабораторная работа № 2: Автоматизация тестирования пузырьковой сортировки

## Описание проекта

Реализован алгоритм **пузырьковой сортировки (Bubble Sort)** с полным набором модульных тестов.

    └── BubbleSortTest.java      # Модульные тесты

## Реализованный функционал

### Класс BubbleSort

Класс содержит два метода:

1. **`sort(int[] array)`** - сортировка по возрастанию
2. **`sortDescending(int[] array)`** - сортировка по убыванию

Особенности реализации:
- Проверка на null с выбросом исключения `IllegalArgumentException`
- Создание копии массива (исходный массив не изменяется)
- Оптимизация: досрочный выход при отсутствии обменов
- Обработка граничных случаев (пустой массив, один элемент)

## Классы эквивалентности входных данных

### Правильные варианты:
1. **Массив с уникальными элементами** - основной случай
2. **Массив с одинаковыми элементами** - проверка на отсутствие изменений
3. **Уже отсортированный массив** - проверка оптимизации
4. **Массив в обратном порядке** - худший случай для алгоритма
5. **Массив с повторяющимися элементами** - проверка стабильности
6. **Массив из одного элемента** - граничный случай
7. **Пустой массив** - граничный случай
8. **Массив с отрицательными числами** - проверка работы с разными значениями

### Ошибочные варианты:
9. **null массив** - должен выбросить исключение

## Принцип AAA (Arrange-Act-Assert)

Все тесты следуют принципу AAA:

```java
@Test
public void sortTestExample() {
    // Arrange - подготовка данных
    int[] array = {5, 2, 9, 1};
    int[] expected = {1, 2, 5, 9};

    // Act - выполнение тестируемого метода
    int[] actual = BubbleSort.sort(array);

    // Assert - проверка результата
    assertArrayEquals(expected, actual);
}
```

## Алгоритм пузырьковой сортировки

### Временная сложность:
- **Лучший случай**: O(n) - массив уже отсортирован
- **Средний случай**: O(n²)
- **Худший случай**: O(n²) - массив отсортирован в обратном порядке

### Пространственная сложность:
- O(n) - из-за создания копии массива

### Принцип работы:
1. Проходим по массиву
2. Сравниваем соседние элементы
3. Если они в неправильном порядке - меняем местами
4. Повторяем, пока массив не будет отсортирован
5. Оптимизация: если за проход не было обменов - массив отсортирован

## Примеры использования

```java
// Сортировка по возрастанию
int[] numbers = {5, 2, 8, 1, 9};
int[] sorted = BubbleSort.sort(numbers);
// sorted = {1, 2, 5, 8, 9}
// numbers = {5, 2, 8, 1, 9} (не изменился)

// Сортировка по убыванию
int[] descending = BubbleSort.sortDescending(numbers);
// descending = {9, 8, 5, 2, 1}
```
